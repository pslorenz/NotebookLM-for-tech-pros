# NotebookLM Notebook Starter Checklist

A checklist for setting up a new NotebookLM notebook for a study topic. Not every step evenly applies to every topic though the sequence is deliberate.

## Naming

- [ ] Notebook named specifically, including the topic and the month or quarter. "Conditional Access Hybrid BYOD 2026-Q2" not "CA Study".
- [ ] If this notebook is part of a series on adjacent topics, the names share a prefix so they sort together.

## Source selection

- [ ] Three to five primary sources identified before opening NotebookLM. Vendor documentation, standards body publications, academic papers, or equivalent.
- [ ] Two to four practitioner or secondary sources identified. Deliberately diverse in perspective or environment.
- [ ] At least one adversary-perspective source identified for any defensive or control topic. Red team writeup, threat intelligence report, or similar.
- [ ] Source count stays under ten for focused study. If the topic legitimately needs more, split into two notebooks.

## Source hygiene

- [ ] Publication date verified for every source. Stale sources flagged or excluded.
- [ ] Sources that are derivative of other sources in the set identified and one is removed.
- [ ] At least one source disagrees with or critiques the consensus. If all sources agree, the study will produce a one-sided understanding.

## Initial ingestion

- [ ] The single best primary source ingested first. A briefing generated from that one source to establish baseline vocabulary.
- [ ] The remaining sources ingested in order of decreasing authority. Briefing regenerated.
- [ ] Study guide generated and reviewed. Questions marked up for which ones you cannot yet answer.
- [ ] Mind map generated. Thin branches noted as potential source gaps.

## First queries

- [ ] Source inventory query run. Confirms NotebookLM correctly identified what you gave it.
- [ ] Disagreement analysis query run. Surfaces tension between sources.
- [ ] Configuration extraction query run if the topic has specific technical parameters.
- [ ] Unasked questions query run. Surfaces blind spots.

## Notes and promotion

- [ ] At least one useful query answer saved as a note.
- [ ] At least one note promoted back to a source, so subsequent queries benefit from the synthesis.

## Before moving to Claude

- [ ] You can describe the topic at a conceptual level in your own words.
- [ ] You have identified at least two areas where your understanding is thin.
- [ ] You have a reference card of specific technical values extracted from the sources.

If you cannot check these last three, stay in NotebookLM and run more queries. Moving to Claude too early produces weaker learned outcomes.
