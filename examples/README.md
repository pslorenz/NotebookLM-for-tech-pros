# Worked Examples

Two worked study sessions, each demonstrating the full Learning Loops method on a different kind of topic.

## The two examples

**`laps-study.md`** walks through a ninety-minute session on Windows LAPS, a discrete feature with a defined configuration surface. The session compresses a topic the practitioner did not previously know well into a deployable understanding in a single afternoon. The `laps-deliverables/` directory contains the four artifacts produced from that session: a practitioner synthesis, a deployment runbook, a detection rule concept, and a lab guide with instructor key.

**`conditional-access-study.md`** walks through a session on Conditional Access policy architecture, an architectural pattern rather than a discrete feature. The session refines a topic the practitioner already knows deeply, and surfaces specific errors, feature surface changes, and architectural assumptions that the practitioner had not recently stress-tested. The deliverables for this example are described structurally rather than reproduced, because they would be substantially similar to client work the practitioner produces regularly.

## Why two examples in different shapes

The first example shows the method working on an unfamiliar topic. The second shows the method working on a familiar one. The outputs look different in each case, and the study time is spent differently, but the method is the same.
